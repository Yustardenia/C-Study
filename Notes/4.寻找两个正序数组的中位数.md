# 4.寻找两个正序数组的中位数

## 题目类型

- 数组
- 双指针
- 有序数组合并

## 我的第一反应

- 想直接把两个数组合成一个新数组，再去取中位数。
- 但是卡在“两个有序数组怎么缝合”这一步，没有立刻想到双指针。
- 还会被题目的 `O(log(m+n))` 要求影响，导致思路发散。

## 我的难点

- 太久没写算法题，看到题目后没能先拆成更小的问题。
- 没有第一时间把问题转换成“两个有序数组如何像拉链一样合并”。
- 容易一开始就盯着最优解，没有先写一个朴素可运行的版本恢复手感。

## 我这次的错误

- 只处理了两个数组都还有元素时的比较逻辑，没有处理某个数组先结束后，另一个数组剩余元素的拷贝。
- 默认 `nums3` 已经被完整填充，实际上会留下未赋值位置，导致中位数计算错误。
- 偶数长度时写成了整数除法，丢失了小数部分。
- 写了 `maxlength` 但没有实际使用，说明思路当时还不够收敛。

## 正确思路

- 第一步：用双指针合并两个有序数组。
- 第二步：如果某个数组先遍历结束，把另一个数组剩余元素补进结果数组。
- 第三步：根据总长度奇偶性取中位数。
- 第四步：朴素解写通以后，再考虑如何优化到不真正合并整个数组。

## 复杂度

- 时间复杂度：`O(m + n)`
- 空间复杂度：`O(m + n)`

## 复盘

- 以后遇到这类题，先不要被最优复杂度要求卡住，先写朴素解。
- 看到两个有序数组，先联想到双指针。
- 写完主体循环后，要立刻检查“剩余元素是否处理完”。
- 涉及平均值时，要警惕整数除法。

## 最优解

- 对这题来说，当前最优解记录标准二分切分法。它比朴素合并更难，但这是题目要求的目标复杂度，且模型固定，值得反复回顾。

```csharp
public class Solution
{
    public double FindMedianSortedArrays(int[] nums1, int[] nums2)
    {
        if (nums1.Length > nums2.Length)
        {
            return FindMedianSortedArrays(nums2, nums1);
        }

        int m = nums1.Length;
        int n = nums2.Length;
        int left = 0;
        int right = m;

        while (left <= right)
        {
            int i = (left + right) / 2;
            int j = (m + n + 1) / 2 - i;

            int nums1Left = (i == 0) ? int.MinValue : nums1[i - 1];
            int nums1Right = (i == m) ? int.MaxValue : nums1[i];
            int nums2Left = (j == 0) ? int.MinValue : nums2[j - 1];
            int nums2Right = (j == n) ? int.MaxValue : nums2[j];

            if (nums1Left <= nums2Right && nums2Left <= nums1Right)
            {
                if ((m + n) % 2 == 1)
                {
                    return Math.Max(nums1Left, nums2Left);
                }

                return (Math.Max(nums1Left, nums2Left) + Math.Min(nums1Right, nums2Right)) / 2.0;
            }

            if (nums1Left > nums2Right)
            {
                right = i - 1;
            }
            else
            {
                left = i + 1;
            }
        }

        return 0;
    }
}
```
