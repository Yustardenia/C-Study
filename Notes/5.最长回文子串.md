# 5.最长回文子串

## 题目类型

- 字符串
- 双指针
- 中心扩展

## 我的第一反应

- 想从字符串两边往中间夹。
- 但这种“整体左右夹逼”更适合判断一个串是不是回文，不适合找最长回文子串。
- 后来改成了“以某个位置为中心，向两边扩”的思路。

## 我的难点

- 容易把“从两边夹”和“从中心往两边扩”混为一谈。
- 容易把当前中心 `now` 和扩展时使用的 `left/right` 写乱。
- 对 `Substring(start, length)` 还不够熟，容易把第二个参数当成右边界。
- 没有第一时间意识到回文分为奇数长度和偶数长度两种中心。

## 我这次的错误

- 用 `now--`、`now++` 直接参与左右指针的赋值，导致中心点和边界都混乱。
- 在内层扩展循环里修改 `now`，导致循环控制变量失真，甚至可能死循环。
- 在字符不相等时，直接用当前的 `left/right` 作为回文边界，但此时这两个位置已经不是合法回文区间。
- 只考虑了奇数回文中心，没有正确覆盖偶数回文。
- `Substring` 的第二个参数误写成了右边界，而不是长度。

## 正确思路

- 这题不是从字符串最外侧往里夹，而是枚举每一个可能的回文中心。
- 对每个中心做两次扩展：
  - `Expand(i, i)` 处理奇数回文。
  - `Expand(i, i + 1)` 处理偶数回文。
- 只要左右字符相等，就继续往两边扩。
- 扩展过程中持续更新当前最长回文的起点和长度。
- 最后用 `Substring(start, maxLen)` 返回结果。

## 复杂度

- 时间复杂度：`O(n^2)`
- 空间复杂度：`O(1)`

## 复盘

- 以后看到“回文子串”，优先想到“中心扩展”。
- “从两边夹”要先问自己：是从整个串两端夹，还是从某个中心向两边扩。
- 只要涉及 `Substring`，就提醒自己参数是 `(起点, 长度)`。
- 写字符串双指针题时，循环控制变量和左右扩展变量不要混用。

## 最优解

- 对你当前阶段来说，最优解记录“中心扩展法”。它不是理论上最强的 Manacher，但最好理解、最好复现，面试和刷题都足够常用。

```csharp
public class Solution
{
    public string LongestPalindrome(string s)
    {
        if (s.Length < 2)
        {
            return s;
        }

        int start = 0;
        int maxLen = 1;

        for (int i = 0; i < s.Length; i++)
        {
            Expand(s, i, i, ref start, ref maxLen);
            Expand(s, i, i + 1, ref start, ref maxLen);
        }

        return s.Substring(start, maxLen);
    }

    private void Expand(string s, int left, int right, ref int start, ref int maxLen)
    {
        while (left >= 0 && right < s.Length && s[left] == s[right])
        {
            int len = right - left + 1;
            if (len > maxLen)
            {
                start = left;
                maxLen = len;
            }

            left--;
            right++;
        }
    }
}
```
