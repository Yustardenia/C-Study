# 6.Z字形变换

## 题目类型

- 字符串
- 模拟
- 按行构造

## 我的第一反应

- 一开始不太容易直接想出最终字符串的规律。
- 如果硬去画完整的 Z 字形，再一列一列处理，会把问题想复杂。
- 这题更自然的做法其实不是“还原整个图形”，而是“模拟字符落到哪一行”。

## 我的难点

- 不容易第一时间意识到只需要按行收集字符，不需要真的建一个二维矩阵。
- 容易在“向下走”和“向上走”之间切换时把行号写乱。
- 容易漏掉特殊情况：
  - `numRows == 1`
  - `numRows >= s.Length`

## 我这次自己写时暴露出的错误

- 把 `nowRow` 和 `goDown` 写进了 `foreach` 循环内部。
  - 这样每处理一个字符，它们都会重新变回初始值。
  - 结果就是状态无法延续，相当于每个字符都“重新从第 0 行开始”。
- 把 `rows[nowRow].Append(c)` 写成了 `rows[c].Append(c)`。
  - `c` 是字符，不是行号。
  - `rows[...]` 需要的是“当前应该写入哪一行的下标”，所以只能用 `nowRow`。
- 对“状态变量”和“当前字符变量”的角色区分还不够稳定。
  - `c` 负责表示“现在处理的是哪个字符”。
  - `nowRow` 负责表示“这个字符应该被放到哪一行”。
  - 这两个变量含义完全不同，不能混用。
- 已经想到了方向切换的判断，但状态位置放错后，整段逻辑就失效了。
  - 说明这题你不是完全没思路。
  - 真正的问题是“核心状态要定义在循环外并持续更新”这个点还不够牢。

## 我这次为什么不会

- 还没有把“模拟路径”这种字符串题型建立起条件反射。
- 看到 Z 字形会先被图形干扰，容易去想坐标、矩阵、列数这些更绕的东西。
- 对“方向切换”类题目还不够熟，没能立刻抓住核心状态：
  - 当前在哪一行
  - 当前是向下还是向上

## 正确思路

- 准备 `numRows` 个字符串，每个字符串代表一行。
- 从左到右遍历原字符串 `s`，把当前字符放进当前行。
- 用一个变量 `currentRow` 表示当前所在行。
- 用一个布尔变量 `goingDown` 表示当前方向：
  - `true` 表示向下
  - `false` 表示向上
- 每放入一个字符后：
  - 如果当前到了第 `0` 行，接下来必须向下
  - 如果当前到了第 `numRows - 1` 行，接下来必须向上
- 最后把所有行拼接起来，就是答案。

## 为什么这样想

- 题目最后要的不是 Z 字形图本身，而是“按行读出”的结果。
- 既然最终只关心每一行收集到了哪些字符，那就直接维护每一行。
- 这样可以把复杂的图形问题，转成简单的遍历模拟问题。

## 特殊情况

- 如果 `numRows == 1`，根本不会形成 Z 字形，直接返回原字符串。
- 如果 `numRows >= s.Length`，每个字符最多单独占一行，也直接返回原字符串。

## 复杂度

- 时间复杂度：`O(n)`
- 空间复杂度：`O(n)`

## 复盘

- 以后看到这种“按某种轨迹摆放字符，再按某种顺序读出”的题，优先考虑“模拟过程”。
- 先想清楚最小状态，而不是先想整个图怎么画出来。
- 这题的最小状态只有两个：
  - 当前行 `currentRow`
  - 当前方向 `goingDown`
- 以后只要题目依赖“上一轮的结果影响下一轮”，就先警惕：
  - 这个变量是不是应该放在循环外？
  - 它是不是整个过程共享的状态？
- 写数组下标时也要先问自己：
  - 我现在放进去的是“字符”还是“位置”？
  - `rows[...]` 里面应该是行号，不应该是字符本身。

## 这次错在哪一行

```csharp
foreach (var c in s)
{
    int nowRow = 0;
    bool godown = false;
```

- 错误点：
  - `nowRow` 和 `godown` 不能定义在这里。
- 原因：
  - 这里只要进入下一轮循环，就会重新赋值。
  - 这样无法保留上一轮走到了哪一行、当前方向是什么。

```csharp
rows[c].Append(c);
```

- 错误点：
  - 下标写成了 `c`。
- 原因：
  - `c` 是字符，表示当前处理到的字母。
  - 这里应该用 `nowRow`，表示当前字符要放到哪一行。

## 这次最该记住的话

- `c` 是“字符”。
- `nowRow` 是“位置”。
- `goDown` 是“方向”。
- 位置和方向是整个遍历过程共享的状态，所以必须写在循环外。

## 可直接背的写法

```csharp
public class Solution
{
    public string Convert(string s, int numRows)
    {
        if (numRows == 1 || numRows >= s.Length)
        {
            return s;
        }

        StringBuilder[] rows = new StringBuilder[numRows];
        for (int i = 0; i < numRows; i++)
        {
            rows[i] = new StringBuilder();
        }

        int currentRow = 0;
        bool goingDown = false;

        foreach (char c in s)
        {
            rows[currentRow].Append(c);

            if (currentRow == 0 || currentRow == numRows - 1)
            {
                goingDown = !goingDown;
            }

            currentRow += goingDown ? 1 : -1;
        }

        StringBuilder result = new StringBuilder();
        foreach (StringBuilder row in rows)
        {
            result.Append(row);
        }

        return result.ToString();
    }
}
```

## 一句话记忆

- 不用真的画 Z，只要按“向下 / 向上”模拟，把字符丢进对应行里。
