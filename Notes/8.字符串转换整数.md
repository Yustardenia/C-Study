# 8.字符串转换整数 (atoi)

## 题目类型

- 字符串
- 模拟
- 边界判断

## 我的当前状态

- 这题我已经能写出通过版本。
- 说明我已经掌握了这题的主流程：
  - 跳过前导空格
  - 处理符号
  - 连续读取数字
  - 遇到非数字停止
  - 溢出时截断到边界
- 现在更需要做的是把代码整理得更清晰，形成稳定模板。

## 我这版通过解法的优点

- 用 `isFirst` 控制“当前还处在前导阶段”，思路是通的。
- 能正确跳过前导空格。
- 能正确识别 `+` 和 `-`。
- 遇到非数字时能及时停止。
- 能在累积过程中处理正负号。
- 能在溢出时返回 `int.MaxValue` 或 `int.MinValue`。

## 我这版可以优化的点

- `Int32.Parse(c.ToString())` 没必要。
  - 这里直接写 `c - '0'` 更简单，也更快。
- `foreach + isFirst` 能写通，但这题本质是“扫描字符串”，用下标 `i` 会更自然。
- 负数边界判断虽然能过，但可读性不如“统一保存正数部分，最后乘符号”的写法直观。

## 我这版已通过代码

```csharp
using System.Text;

public class Solution {
    public int MyAtoi(string s)
    {
        int ans = 0;
        bool isPos = true;
        bool isFirst = true;
        int x = 0;
        foreach (var c in s)
        {
            if (c == ' ' && isFirst)
            {
                continue;
            }

            if (c == '+' && isFirst)
            {
                isFirst = false;
                continue;
            }

            if (c == '-' && isFirst)
            {
                isPos = false;
                isFirst = false;
                continue;
            }

            if (c > '9' || c < '0')
            {
                break;
            }

            else
            {
                x = Int32.Parse(c.ToString());
                isFirst = false;
            }
            if (ans > Int32.MaxValue / 10 || (ans == Int32.MaxValue / 10 && x > 7))
            {
                return Int32.MaxValue;
            }

            if (ans < Int32.MinValue / 10 || (ans == Int32.MinValue / 10 && x > 8))
            {
                return Int32.MinValue;
            }

            ans = ans * 10 + (isPos ? x : -x);
        }
        return ans;
    }
}
```

## 更推荐记住的模板写法

```csharp
public class Solution
{
    public int MyAtoi(string s)
    {
        int i = 0;
        int n = s.Length;

        while (i < n && s[i] == ' ')
        {
            i++;
        }

        int sign = 1;
        if (i < n && (s[i] == '+' || s[i] == '-'))
        {
            sign = s[i] == '-' ? -1 : 1;
            i++;
        }

        int result = 0;

        while (i < n && char.IsDigit(s[i]))
        {
            int digit = s[i] - '0';

            if (result > int.MaxValue / 10 ||
                (result == int.MaxValue / 10 && digit > (sign == 1 ? 7 : 8)))
            {
                return sign == 1 ? int.MaxValue : int.MinValue;
            }

            result = result * 10 + digit;
            i++;
        }

        return result * sign;
    }
}
```

## 为什么这个模板更清晰

- 它和题目描述的步骤一一对应：
  - 先跳空格
  - 再读符号
  - 再读数字
  - 每轮判断溢出
- `i` 明确表示当前扫描位置，比 `foreach` 更适合这种分阶段处理题。
- `digit = s[i] - '0'` 比 `Parse` 更直接。
- `result` 始终保存“数字部分”，最后再统一乘上 `sign`，整体更好理解。

## 溢出判断怎么理解

题目核心仍然是这一句：

```csharp
result = result * 10 + digit;
```

但在真正执行前，要先判断这一步会不会越界。

```csharp
if (result > int.MaxValue / 10 ||
    (result == int.MaxValue / 10 && digit > (sign == 1 ? 7 : 8)))
{
    return sign == 1 ? int.MaxValue : int.MinValue;
}
```

- 如果 `result` 已经大于 `214748364`，再乘 `10` 一定越界。
- 如果 `result` 正好等于 `214748364`：
  - 正数最后一位最多只能是 `7`
  - 负数最后一位最多可以到 `8`
- 因为：
  - `int.MaxValue = 2147483647`
  - `int.MinValue = -2147483648`

- 这题本质是“字符串扫描模板题”。
- 以后看到这种题，优先按阶段拆：

  - 读符号
  - 读数字
