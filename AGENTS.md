# AGENTS.md

本目录用于编写和整理 LeetCode 算法题代码与笔记。

## 目标

- 优先把题做出来，再考虑优化。
- 优先训练稳定的解题流程，而不是追求一次写出最优解。
- 每道题保留代码、思路、错误记录和复盘内容，方便后续回看。

## 代码约定

- 默认使用 C# 控制台项目。
- 入口文件保留显式 `Program` 类，不使用顶级语句。
- 题解代码优先写成 `Solution` 类中的方法，贴近 LeetCode 提交格式。
- 如果只是本地测试，可以在 `Program.Main` 中临时调用；测试代码应尽量简短，完成后可以删除或注释。
- 先写朴素解，再写优化解；如果有多个版本，用注释标明 `朴素解`、`优化解`。

## 目录约定

- `Program.cs`：本地测试入口。
- `Notes/`：统一存放刷题笔记。
- 如后续题目变多，可再按题型或阶段拆分子目录，但笔记仍优先放在 `Notes/` 下。

## 笔记命名规则

- 每道题单独一个 Markdown 文件。
- 文件名格式统一为：`题号.题名.md`
- 示例：
  - `1.两数之和.md`
  - `4.寻找两个正序数组的中位数.md`
  - `206.反转链表.md`

## 错题回顾规则

- 错题总表固定放在 `Notes/1A.错题回顾.md`。
- 只有“最终没独立改对”或“虽然做出但核心思路仍明显不稳、短期内大概率再错”的题，才加入错题回顾。
- 如果一道题最后是你自己独立改对的，则保留单题笔记，但不加入错题回顾。
- 错题回顾优先记录：
  - 为什么会卡住。
  - 哪些错误反复出现。
  - 下次回顾时应该重点看什么。

## 笔记模板

每道题按以下结构记录：

```md

## 我的难点

- 没想到的关键点

- 代码错误

- 思路错误

- 先用最朴素的话说明怎么做
- 再总结关键步骤

## 复杂度

- 时间复杂度：
- 空间复杂度：

## 复盘

- 以后再遇到类似题，应该先想什么
- 这题可以迁移到哪些类似题

- 在“容易理解”和“解题效果”之间做综合取舍，记录当前最值得背下来的版本。

- 如果题目存在更优但明显更绕的做法，可以在这里顺带注明，但主记录版本应以可理解、可复现为先。
```

## 解题流程

每次做题尽量按这个顺序：

1. 先用自己的话重述题意。
2. 先判断能不能写出朴素解。
3. 明确输入、输出、边界条件。
4. 写出核心过程，不要急着优化。
5. 用示例手推一遍。
6. 写代码。
7. 检查边界和返回值类型。
8. 最后再看是否能优化复杂度。
9. 把错误和卡点记进对应题目的笔记文件。
10. 在笔记最后补上“最优解”。

## 常见自查项

- 数组是否越界。
- `while` 循环结束后是否有剩余元素没处理。
- 奇偶长度是否分开处理。
- 是否误用了整数除法。
- 空数组、单元素、重复元素是否考虑到。
- 变量是否定义了但没用；如果有，通常说明思路还没收紧。

## 与助手协作时的默认要求

- 优先给出从朴素解到优化解的过程，不要一开始只给最优解。
- 优先指出“我为什么会卡住”，再给代码。
- 如果我要记录笔记，优先写入 `Notes/题号.题名.md`，保持格式统一。
- 如果涉及是否加入错题回顾，按本文件中的“错题回顾规则”处理。
- 如果我要写某道题，默认先给思路拆解，再给 C# 实现。
- 每道题的笔记最后都要附上“最优解”。
